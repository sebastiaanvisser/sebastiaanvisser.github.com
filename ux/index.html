Palantir
aug 2016 — jul 2019

Please don't share any of this work publicly.

  • Fourier

    Fourier is a grapical user interface for building data pipelines that get executed on the backend in a Python compute kernel. Power users can build pipeline templates by annotating Python functions with input and output types. A custom inference engine in the frontend makes sure only appropriate pipeline steps are suggested to the user and will make sure pipelines are valid by construction. The result data of a pipeline can be written back to the underlying data platform or presented in the UI with special visualization steps.
    Main Fourier screen with pipeline steps on the left and map visualization in the main output window.
    Small experiment with branching pipeline steps using a small indent to indicate structure.
    A mock showing the layout of the Fourier function library. List of functions with basic version control status on the left, code editor in the middle and type/metadata configuration on the right.
    Playing around with the logo design for Fourier, as well as Titan. Titan was a rebrand of Fourier specialized for machine learning pipelines using PySpark+Tensorflow.
    Fourier REPL mode where you can exectute code directly in the context of the active/selected pipeline step. All data in scope, like the output of the input step are available for debugging purposes. Values that match known visualizations steps can be inspected on the fly.
    The workflow editor allows you to create pipeline templates by stiching together functions of the right type and adding the neccesary metadata.

  • Explorer

    Explorer is an experimental table-first table processing tool build around the concept of object types with relations forming knowledge graphs. The engine driving all of the UX is a small typed functional programming language specialized for building graph queries and table manipulation operations.
    The main table UI showing typed values including units for numeric values.
    Every column represents some pivot in the knwoledge graph and adding derived columns is like building a graph query based on a search-around in the meta model.
    Beside the table view showing a collection of object types there is also an object view that zooms in on specific items. Different types of information can have different visual representations.
    The object view at the basis represents a single row from a tabular source. However, different information can be directly pulled in based on context sensitive search-around.
    An alternative map visualization allows plotting everything that associated geo locations, either directly on the entity or accessible via some context query.
    The table view can display hierarchical information by joining in information from other data sources via relations. When multiple results are returned per row a table is nested creating fan-out like behavior.

  • Ontology Editor

    The ontology editor is a small utility on the Foundry data platform allowing users to author the meta model connected to their data storage. It allows configuration of object types, relations and meta information that will drive all the object based UX in the platform. Additionally it allows configuration of more technical details involved in ingesting and enriching raw data into a more semantic form.
    Main ontology editor window as a standalone application. List of editable object types and relations on the left and the editing panel on the right.
    The ontology editor as integrated into the dataset preview application. This allows users to make raw data part of the vetted object world directly after importing.
    By scanning a snapshot of the raw data the ontology editor can auto suggest a lot of the configuration values with sane defaults. Even relations to other object types can be inferred by trying likely join points.
    Object types with both incoming and autogoing relations could be inspected in graph form to gain a better understanding of your data model. Clicking on the relation nodes would scroll and focus that object type so you could walk the graph.
